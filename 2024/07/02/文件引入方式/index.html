link 和 @import

二者均是样式文件的导入方式

link

@import

区别

  • 引入内容
    • link除了引入样式文件，还可以引用图片等资源文件
    • @import只能引用样式文件
  • 加载顺序
    • link引用CSS时，在页面载入时同时加载
    • @import需要页面完全载入之后再加载
  • 兼容性
    • link是HTML标签，无兼容问题
    • @import是CSS2.1提出的，低版本浏览器不支持
  • 对JS的支持
    • link支持使用JavaScript控制DOM改变样式，例如鼠标悬浮时改变颜色
    • @import不支持

href和src获取资源

src

script和img用src获取资源。

  • src是source的缩写，指向外部资源的位置，指向的内容将会嵌入到文档中当前标签所在位置
  • 在请求src资源时，会将其指向的资源下载并应用到文档内

当浏览器解析到该元素时，会暂停其他资源的下载和处理，直到该资源加载、编译、执行完毕。因此，应该将js脚本放在底部。

href

link用href获取资源。

  • href是Hypertext Reference的缩写，指向网络资源所在位置，建立和当前元素（锚点）或当前文档（链接）之间的链接
  • 在文档中添加link标签，浏览器会识别该文档为css文件，并行下载资源并且不会停止对当前文档的处理。因此建议使用link方式加载css，而不是@import
